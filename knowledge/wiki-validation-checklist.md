# Wiki監査チェックリスト — P1-P4タスク・推測タイトル検証・バリデーション手順

## TL;DR

Wiki運用で最大の脅威は **推測タイトルの無検証化**。旧「電力系統・需給運用」がAI推測で放置されていた例から、全セクション・全Wikiに監査が必須。P1-P4タスクで段階的に修正。バリデーション手順は「電験王3 URL確認 → セクション再分類 → ページタイトル正題化」の3段階。

## 1. 監査スコープ定義（P1-P4）

### P1: 電力Wiki `themes/denryoku-keitou.md` ハブの再定義（最優先）

**背景**: by-field.md から「電力系統・需給運用」セクション削除 → ハブページ（themes/）がリンク切れ状態の可能性

**検証項目**:
1. 電験王3 で実際に「電力系統運用」が出題されている問番号を特定（問17・18等？）

3. ハブページを立て直すか削除するか判定

**実行コマンド**:

**工数**: 2時間（全問スポット確認 + 再分類）

## 2. 推測タイトル検出パターン

### パターンA: AI推測が明らかな場合

**特徴**:
- セクション名が「〇〇運用」「〇〇管理」で終わる（generic な表現）
- 電験王3 の実際の問タイトルと一致しない
- 複数年度でセクション名がゆらぐ

**例**: 旧「電力系統・需給運用」
- 実際の電験王3には存在せず、by-field.md の AI推測で挿入
- 実際の出題は「送配電線路」か「その他」に分類される

**検出フロー**:
```
1. セクション名のGeneric度チェック：「運用」「管理」「○○系統」で終わるか？
2. 電験王3 URL で該当年度の全問をスクロール確認
3. 差分あれば、該当セクションを[要確認]マークして pending
4. スポット確認 → 正式タイトルに置き換える
```

### パターンB: サブセクション重複の可能性

**特徴**:
- 変圧器 と 電気施設管理 が「全日効率計算」で重複
- 2つのエンティティが同じトピックをカバーしているが、どちらが主体か未定義

**検出フロー**:
```
1. 各エンティティの frontmatter を確認
2. related: に相互リンクあるか？→ なければ追加
3. 内容重複の場合は統合コメント追加（「参照: [[別エンティティ]]」）
```

## 3. バリデーション手順（3段階）

### 段階1: 電験王3での URL確認

**目標**: 実際のセクション名・問タイトルを取得

**方法**:
```
URL形式（電力の例）:
- R04-R07上下期: https://denken-ou.com/denryokuR{N}-{1|2}-{問番号}/
- R01-R03上下期なし: https://denken-ou.com/denryokuR{N}-{問番号}/

科目プレフィックス:
- 電力: denryoku
- 理論: riron
- 法規: houki
```

**実行**:
1. 検証対象セクションの問番号リスト作成
2. 各問について URL を開く（ブラウザ or claude-code fetch）
3. ページ上の「セクション名」「正式タイトル」を記録

**出力**: 差分テーブル
```
| 年度 | 問番号 | 旧セクション（by-field.md） | 電験王3実際 | ステータス |
|-----|--------|------------|---------|----------|
| R04上 | 問10 | 送配電線路 | 送配電線路 | OK |
| R03 | 問11 | 電力系統・需給運用 | ？（要確認） | 要修正 |
```

---

### 段階2: セクション再分類（影響範囲確認）

**目標**: 各問がどのセクション（電験王3の実際の括り）に属すべきか確定

**検証項目**:
1. **セクションの統合判定**: 旧「電力系統・需給運用」が実在するか？or 複数セクションに散在するか？
2. **問の移動**: 問を正しいセクションに再割当
3. **リンク整合性**: by-field.md と themes/ の相互リンク確認

**例**: 「電力系統・需給運用」の判定結果パターン
- **パターンA**: 実在する正式セクション → そのままセクション名に採用
- **パターンB**: 実在しない → 複数の実セクション（例「送配電線路」「火力発電」）に分割
- **パターンC**: サブテーマ → 親セクションに統合

---

### 段階3: ページタイトル正題化（wiki ページの修正）

**目標**: wiki/ の entities/ や themes/ のタイトルが電験王3と一致することを確認

**修正フロー**:
1. 各 .md ファイルの frontmatter: `title` を実際の電験王3タイトルに更新
2. 本文の first-heading（# で始まる見出し）も同期
3. related: リンクが正しい entity を指しているか確認

**例**:
```markdown
# 修正前
---
title: "電力系統・需給運用"
---

# 修正後
---
title: "送配電線路と故障点標定"
---
```

## 6. 実践メモ

### 並列エージェント推奨シーン

P2（セクション監査）では複数年度・複数科目の確認が必要 → 落合陽一（体系性検証）+ ひろゆき（工数効率）で並列実行推奨。

### 推測タイトル判定基準

**ホワイトボックス型（確実な正題化可能）**:
- 電験王3 URL で直接確認できた
- 複数年度で同じタイトルで統一

**グレーボックス型（暫定マーク必須）**:
- URL確認したが内容が曖昧
- セクション名にゆれがある
- → `[要確認]` マークを frontmatter に付記

**ブラックボックス型（保留推奨）**:
- 出題実績が不確定
- 推測で入力された可能性あり
- → `published` → `review` に降格、ユーザー判定待ち
