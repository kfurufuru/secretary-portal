# 電験法規学習における認知科学の応用

## TL;DR

電験法規の習得速度は学習方法で**3～5倍**変わる。

- **間隔反復**: Ebbinghaus曲線に従う20分→1時間→1日→1週間スケジュールで忘却を70%カット
- **テスト効果**: 受動読書（42%保持）vs 自己テスト（80%保持）で**2倍の差**
- **インターリービング**: 単一法則の反復（ブロック学習）より異なる法則を混ぜる学習（32～63%改善）が転移・応用力を高める

法規は「条文を丸暗記」ではなく「判定フローの習得」が本質。これらの原則を組み込むと、同じ勉強時間で習得度が劇的に上がる。

---

## 1. Ebbinghaus忘却曲線と間隔反復

### 基本メカニズム

Hermann Ebbinghaus（1885年）の実験に基づく古典的原則：

- **20分後**: 記憶保持率 58%（学習直後から42%忘却）
- **1時間後**: 保持率 44%
- **9時間後**: 保持率 36%
- **1日後**: 保持率 26%（学習から74%忘却）
- **1週間後**: 保持率 23%（復習なしでほぼ消滅）

### 間隔反復スケジュール（最適実装）

復習タイミングを忘却曲線が急落する直前に設定すると、記憶強化が指数関数的に効く：

| 回次 | 推奨タイミング | 脳の状態 | 保持率向上 |
|-----|------------|--------|---------|
| 1回目 | 学習直後 | 当たり前だが必須 | — |
| 2回目 | 20～30分後 | 初忘却の局面で介入 | 70% → 90%に回復 |
| 3回目 | 1日後 | 1回目復習から次の忘却カーブ | 90% → 95%に強化 |
| 4回目 | 1週間後 | 2回目復習からの自然忘却 | 95% → 98% |
| 5回目 | 1ヶ月後 | 長期記憶への定着完了 | 98% → 定着 |

**効果**: 何も対策しない場合と比較して、**1ヶ月後の保持率が26% → 98%に上昇**（同じ勉強時間で）

### 電験法規での適用例

「接地工事の義務者（設置者 vs 施工者）」を学習する場合：

1. **学習直後（0分）**: テキスト読了 → 「設置者が届出義務者」と確認
2. **20分後**: 過去問1問解く → 「施工者ではなく設置者」を再確認
3. **1日後**: 類似の法律（電気工事業法との違い）で比較学習
4. **1週間後**: セクション全体の選択肢混合問題で復習
5. **1ヶ月後**: 模試で確認（定着完了）

## 2. テスト効果（Testing Effect）とリトリーバル・プラクティス

### 実測データ

Rohrer & Taylor（2007年）ほか複数研究の統合メタ分析:

| 学習方法 | 学習直後の成績 | 1週間後の保持率 | 効果サイズ |
|---------|-----------|------------|----------|
| 受動読書（テキスト反復） | 75% | 42% | 基準値 |
| 自己テスト（問題演習） | 65%（直後は低い） | 80% | **+38%** |
| テスト＋間隔反復 | 60%（直後は最低） | 89% | **+47%** |

**つまり**: 学習直後の「わかった感」は信頼できない。むしろ試験本番で点数が取れるなら、テストしながら学ぶ方が2倍有効。

### 脳のメカニズム

- **受動読書**: 短期記憶（ワーキングメモリ）に留まる → 7日後のテストで高確率で忘却
- **自己テスト**: 記憶から「引き出す」という行為が脳の神経経路を強化 → 長期記憶化が促進される

### 電験法規への応用

❌ **やってはいけない学習法**:
```
テキストを読む → 「わかった」と思う → 試験受験
→ 半分以上落とす（理解はしたが再現できない）
```

✅ **効果的な学習法**:
```
テキスト読了 → すぐ過去問を解く → 間違える → 理由を調べる
→ 1日後また同じ問題を解く → 今度は正解 → 1週間後に別パターンで確認
```

**Ebbinghaus + Testing Effect の相乗効果**:

間隔反復スケジュールの各タイミングを「テスト」で埋めると、忘却曲線の回復率が個別より格段に高まる。電験では過去問を「間隔反復の媒体」として使う。

## 3. インターリービング（交互学習）

### 定義と実測結果

**ブロック学習**（一般的）:
```
法規1「接地工事」を10問 → 「届出義務」を10問 → 「主任技術者」を10問
```

**インターリービング**（推奨）:
```
接地・届出・主任技術者の問題を混ぜて10問 × 3セット
```

Rohrer & Hwang（2015年）による実験:

| 学習形式 | 学習段階での成功率 | テスト時の成功率 | 転移タスク成功率 |
|---------|--------------|------------|-------------|
| ブロック学習 | 90% | 35% | 20% |
| インターリービング | 65% | 70% | 65% |

**転移タスク** = 見たことないパターンの問題（試験本番と同じ状況）

**結論**: 学習中は「インターリービングの方が難しく感じる（成功率65%）」が、テスト時には**約2倍**の差が出る（35% vs 70%）。試験本番では問題の順序が原則ぐちゃぐちゃなので、本番環境に合わせた学習が有効。

### 32～63%改善のメカニズム

インターリービングが効く理由は3つ：

1. **区別学習（Discrimination）**: 「接地工事か届出義務か」を毎回判定する習慣が付く
2. **関連知識の同時活性化**: 「接地種別 ← → 届出先」の関連構造が記憶に統合される
3. **転移への準備**: 見慣れない組み合わせ問題に対応できる柔軟性が育つ

### 電験法規での実装例

**ブロック学習（非効率）**:
```
月: 接地工事の過去問 20問
火: 届出義務の過去問 20問
水: 主任技術者の過去問 20問
```
→ 各テーマは得意になるが、テスト時に「この問題は接地か届出か？」の判定で迷う

**インターリービング（効率的）**:
```
月～水: 接地・届出・主任の混合問題 20問ずつ（毎日新しい問題が出るたび、
         「どのテーマか」を判定してから解く）
```
→ 試験と同じ環境で脳が鍛えられる

## 4. デュアルコーディング理論（言語 + 視覚）

### 理論

Paivio（1971年）: 人間の記憶は「言語エンコーディング」と「イメージエンコーディング」の2つのルートで強化される。

| 記憶方式 | 記憶強度 | 例 |
|---------|---------|-----|
| 言語のみ（「接地工事」と読む） | 1.0 | 忘れやすい |
| イメージのみ（接地工事の施工図を眺める） | 1.3 | 中程度 |
| 言語＋イメージ（「接地工事」という言葉 + 接地極の図） | **2.1～2.5** | 忘れにくい |

### 電験法規への応用

- **接地抵抗値の計算**: 数式 + 「接地極本数が増えると抵抗が下がる」イメージ（並列回路図）
- **電圧区分**: 「600V」という数値 + 「低圧/高圧の分界線」を示す図表
- **届出先フロー**: テキストの条件判定 + 「経産大臣 → 都道府県知事 → 産業保安監督部長」の階層図

**実装**: 法規Wikiのセクション5（覚える表）でデュアルコーディングを意識的に設計する。

## 5. メタ学習（学習の学習）

### 定義

自分の学習プロセスを監視・評価し、戦略を改善する高次スキル。

- 「この学習方法で、これだけの時間がかかるか」を予測
- 「このテーマは苦手」と気づいて計画を修正
- 「この解き方は本番で使える」と判断して本番練習に組み込む

### 電験3種の文脈

PMC（過去問メタ認知チェック）の設計:

| メタ認知要素 | 実装 | 効果 |
|-----------|------|------|
| 自信度の事前予測 | 解く前に「これ正解できる？」と予測 | 実的中率が70%超なら、自己評価が精密 |
| 誤答分析 | 「計算ミス」「読み間違い」「知識欠落」を分類 | 同じ誤りを繰り返さない |
| 学習計画の修正 | 「苦手な条文 = 3日ごとに復習」と決める | 時間配分が最適化される |

## 6. 電験法規学習に特化した戦略

### 「選択肢全体を覚える」の陥穽

法規は「A～Dの4選択肢を丸暗記」では、応用が効かない。なぜなら：

- 過去問と同じ選択肢は出ない
- 「接地工事」と「接地抵抗値」が組み合わさった新問題が出る
- 正解選択肢を説明できなければ、出題者が前年と微妙に変えた問題で落とされる

**正しいアプローチ**:
1. **選択肢の判定フローを習得**: 「この問題で問われてるのは何か」を5秒で判定
2. **根拠条文を言える**: 「なぜAではなく B が正解か」の理由を条文番号で説明
3. **類推力を鍛える**: 見たことない選択肢に出会っても判定できる思考プロセス

### ゴール問題の役割

Ambrose et al.（2010年）: 「学習の最初に『これが解けるようになるのがゴール』を示す」と、学習中の脳が目標志向的に情報を処理する。

- 単なる「わかる」ではなく「この問題を解ける」状態への焦点が定まる
- 学習完了後に同じ問題を解き直すと、「理解 → 実行力への変換」を確認できる
