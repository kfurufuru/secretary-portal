# ターミナル・シェル・CLIの違い

**TL;DR**: 3つは役割が異なる別物。ターミナル=窓口(表示)、シェル=解釈エンジン(実行)、CLI=操作スタイル(概念)。

## ポイント

- **ターミナル（Terminal）**: 画面を表示するアプリケーション（ウィンドウ）。文字・色・フォント・キーボード入力を担う。例: Windows Terminal, iTerm2, PuTTY
- **シェル（Shell）**: ターミナルの中で動くプログラム。コマンドを解釈・実行する。変数・パイプ・リダイレクト・スクリプトも提供。例: Bash, Zsh, Fish, PowerShell
- **CLI（Command Line Interface）**: 「コマンドを入力して操作する」というインターフェーススタイル（概念）。特定プログラムではない。例: Git CLI, Docker CLI, AWS CLI

## 連携の流れ

```
あなた（コマンド入力）
  → ターミナル（入力を受け取り表示）
  → シェル（コマンドを解釈・実行）
  → CLIプログラム（例: git, docker, ls）（実際の処理）
  → 実行結果がターミナルに返されて表示
```

## 詳細

| 項目 | ターミナル | シェル | CLI |
|------|-----------|--------|-----|
| 正体 | アプリ（ウィンドウ） | プログラム | 概念・スタイル |
| 役割 | 表示・入出力 | 解釈・実行 | システム・アプリへの命令 |
| 具体例 | Terminal, xterm, PuTTY | bash, zsh, fish | git, docker, aws |
| 使い方 | 開く（起動） | コマンド入力 | コマンドで操作 |

## 落とし穴

- **「ターミナル=シェル」と混同しやすい**。ターミナルはあくまで窓。シェルがその中で動いている。
- CLIは「コマンドプロンプト」「ターミナル」と呼ばれることもあるが、正確には「操作方法のスタイル」。
- PowerShellはシェルだが、Windows環境では「CLIの窓口」として語られることも多い（ターミナル兼用）。
