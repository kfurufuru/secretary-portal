## 今回の学び（重要）

1. **監修プロセスの「PASS」が数値だけ照合の事故パターン**: 56.md(2026-05-02 11A→27A誤記) の対策で§7.3 R1〜R5 を整備したが、今回新たな型の事故を発見。**条文番号・タイトル・本文の三点照合が必須**。R6 として追加すべき。

2. **ローカルPDF Read 戦略は強力**: 電技省令本体（211KB・78条）を1回 Read で全条文一括取得 → 11条文を本体並列 Edit で処理。WebFetch 経由なら30分はかかる作業が5分で終わった。CLAUDE.md #14b 「ローカル資産優先探索」が機能。

3. **kakomon.yml の限界**: `topic`（論点）レベルまでしか持たない。実際の穴埋め空欄（どの単語が抜かれたか）は別ソース（電験王3 等）が必要。今回は仮置きで「※実空欄は要再確認」フラグを立てた。

4. **規約違反の検出は grep で高速**: `grep "^## .*条文原文" docs/articles/kijun/*.md` でセクション欠落11件を即特定。スタイルガイド §2.1 の必須セクションは定期監査が有効。

5. **`§` を含む正規表現は注意**: grep で `§(2|4|...)` だけでは `§22` `§28` も誤ヒットする。`§N[ ,"]` で区切り文字を含めるか、行ごとに sed で抽出する必要あり。
