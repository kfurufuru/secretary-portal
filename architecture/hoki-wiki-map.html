    📝 修正対象ファイル（Source / Data）

    hoki-components.jsx hoki-pages.jsx hoki-data.js hoki-glossary.jsx hoki-glossary-data.js hoki-head-template.html denken-hoki-wiki.html ＝編集禁止
    • 共通UIは hoki-components.jsx（App/Sidebar/QuickReview/MetaStrip 等）
    • ページ本体は hoki-pages.jsx（39 *Page + renderPage）
    • ナビ/章/頻度は hoki-data.js（WIKI_DATA）

    🔍 整合性チェック（push前）

    1. python hoki_check.py
      ＝ pages.jsx ↔ hoki-data.js ↔ kakomon.yml の id/cta/tag/yr
    2. 未確認警告：kakomon.yml の現行パスが script ごとに違う 可能性あり
      → grep -r kakomon.yml で配置確認
    3. JSX構文エラー時は build-hoki-wiki.py の node --check で位置特定

    🏗️ ビルド（build-hoki-wiki.py）

    1. node --check で hoki-data.js / hoki-glossary-data.js 構文検証
    2. hoki-head-template.html の CSS を head に注入
    3. node -e で WIKI_DATA / GLOSSARY を eval → 検索 index 構築
    4. data/hoki-search-index.json 出力 ＋ window.HOKI_SEARCH_INDEX 埋込
    5. 既存 denken-hoki-wiki.html を .bak へ退避
    6. head + CDN + body + ranking + searchIndex + script(babel) を連結出力

    🚀 デプロイ＆反映確認

    1. git add -A && git commit -m "..." && git push
    2. GitHub Pages 自動デプロイ（~50秒）
    3. 確認 URL: denken-hoki-wiki.html
    4. キャッシュ強制更新は ?v=YYYYMMDD 付与で確認
    5. SPA route は #pageid:anchor 形式（: で分離）

    💾 localStorage キー一覧

    • denken_check::hoki_<slug>::<DJB2 hash> — QuickReview 4ボタン状態
    • hoki_lastSeen_<pageId> — 最終閲覧 timestamp
    • hoki_sidebar_mode / hoki_sidebar_window — テーマ別/章別タブ＋集計 5y/10y/15y
    • hoki_quiz_glossary_* — 用語クイズ進捗 ＋ バックアップ
    ※同一オリジン（kfurufuru.github.io）配下で denken_check::* は denken-wiki と 共有。MachigaiNotePage が両方を横断スキャン。

    📊 ランキング更新（任意）

    1. denken-wiki/_data/kakomon.yml 更新を検知
    2. python scripts/gen_hoki_ranking.py を実行
    3. data/hoki-theme-ranking.json（S/A/B/C 4段階）が再生成される
    4. 次回 build 時に window.HOKI_RANKING として埋込
    5. Sidebar のテーマ別タブで 5y/10y/15y ウィンドウ切替表示
