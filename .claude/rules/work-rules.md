# Claude Code 作業ルール

このプロジェクトは、電験3種学習者向けのWikiを作成・改善するためのものです。
目的は、正確で、試験対策に役立ち、初学者にも読みやすいページを作ることです。

## 1. 基本方針

- 回答は前置きせず、結論から始めること。
- 不明点・不確実な点は推測で補完しないこと。
- 法令、制度、数値条件、条文番号、試験頻出論点は特に慎重に扱うこと。
- 情報の正確性が必要な場合は、経産省、e-Gov、電気技術者試験センターなどの正式情報を優先して確認すること。
- ChatGPTアドバイスを確認して、対応が必要なものは修正して。

## 2. スコープ厳守

- 私が依頼した範囲だけを変更すること。
- 関係ないファイル、関係ないセクション、関係ない文言は変更しないこと。
- 「ついでに改善できそう」と思っても、勝手に変更しないこと。
- 追加改善案がある場合は、実装せずに「提案」として最後に分けて書くこと。

## 3. 大きな変更の前に確認

以下の変更を行う前には、必ず一度止まって説明すること。

- ページ構成を大きく変える
- 既存セクションを削除する
- 見出し構成を大きく並べ替える
- 複数ファイルを横断して変更する
- 既存の説明方針や学習導線を変える
- build、deploy、公開、外部送信などの操作を行う

説明するときは、以下を示すこと。

- 何を変えるのか
- なぜ変えるのか
- 影響するファイル
- 元に戻せるかどうか

私の明示確認なしに実行しないこと。

## 4. 最小変更を優先

- まずは最小限の修正で目的を達成すること。
- 大規模リファクタより、試験対策上の誤解を減らす修正を優先すること。
- UI改善も、学習効果に直結するものを優先すること。
- 装飾や見た目の改善だけで本文の正確性を崩さないこと。

## 5. 電験3種Wikiとしての確認観点

修正時は、必ず以下を確認すること。

- 電験3種の受験者にとって得点につながるか
- 初学者が誤解しやすい表現になっていないか
- 条件、例外、対象範囲、主語が明確か
- 「届出」「許可」「認可」「承認」「報告」「通知」を混同していないか
- 数値条件、未満・以下、以上・超過の境界が正しいか
- 計算問題では、単位、前提条件、途中式、選択肢との対応が正しいか
- 法規ページでは、正式情報に照らして誤りがないか

## 6. ChatGPTレビュー反映ルール

ChatGPTからレビュー・改善提案・指摘がある場合は、以下の手順で対応すること。

1. ChatGPTアドバイスを確認する
2. 指摘を以下に分類する
   - 必ず修正すべき誤り
   - 修正した方がよい改善点
   - 参考意見に留めるもの
   - 対応不要なもの
3. 必ず修正すべき誤りから優先して対応する
4. 対応不要と判断したものは、理由を明記する
5. 勝手に大規模改修せず、依頼範囲内で反映する

## 7. 作業後の報告

作業完了後は、必ず最後に以下を簡潔にまとめること。

- 変更したファイル
- 変更した内容
- 変更しなかった箇所
- 確認が必要な点
- build/test結果
- 次に対応した方がよい提案

## 8. MEMORY.md 運用

重要な判断をした場合は、MEMORY.mdに記録すること。

記録する内容は以下。

- 決定したこと
- その理由
- 採用しなかった案
- 次回以降に引き継ぐ注意点

ただし、軽微な文言修正や一時的な作業ログは記録しなくてよい。

## 9. ERRORS.md 運用

同じ修正で2回以上失敗した場合、または原因調査に時間がかかった場合は、ERRORS.mdに記録すること。

記録する内容は以下。

- 何がうまくいかなかったか
- 原因
- 最終的にうまくいった方法
- 次回同じ失敗を避けるための注意点

## 10. 禁止事項

- 根拠のない法令説明を追加しない
- 試験対策上、誤解を招く簡略化をしない
- 依頼外のファイルを勝手に変更しない
- デザイン改善のために本文の意味を変えない
- buildエラーを隠さない
- 不確実なことを断定しない
- 私の確認なしにdeploy・publish・外部送信をしない
