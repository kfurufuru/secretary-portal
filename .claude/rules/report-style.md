# 報告書・メール デザインシステム

## ライティングルール

```
tone:        direct       # 断定調。推量・遠回し禁止
structure:   BLUF         # 結論を必ず冒頭に置く
evidence:    required     # 結論には必ず根拠を添える
numbers:     mandatory    # コスト・期間・件数・効果は必ず数値化
ambiguity:   forbidden    # 「なるべく」「できれば」「早めに」は使わない
length:      minimal      # A4半ページ以内。超える場合は別添え
bullets:     preferred    # 散文より箇条書きを優先
passive:     avoid        # 受け身表現・サ変過多（させていただく）禁止
```

## 相手別スタイル

| 相手 | 冒頭 | トーン | 特記 |
|------|------|--------|------|
| 部下 | Why先行（なぜやるか） | 問い返し型（「どう思う？」を添える） | 1項目1アクション |
| 上司・役員 | 結論1行目 | 断定。リスク明記、過度な謝罪なし | コスト・期間・効果を3点セット。本文10行以内 |
| 製造課 | 影響論先行（停止時間） | 相手コスト優先 | 日程・手順を明示 |
| メーカー・社外 | 背景→要求仕様→回答期限 | 形式的・明確 | 曖昧表現完全禁止 |

## 禁止表現

| 禁止 | 代替 |
|------|------|
| させていただきます | します／行います |
| なるべく早く | YYYY-MM-DDまでに |
| ご検討ください | 〇月〇日までに回答ください |
| ご確認いただけますでしょうか | 確認をお願いします |
| 〜と思われます | 〜です／〜と判断します |
| 努力します | 〇〇します（不確かなら条件付きで明記） |
| ふわっとした励まし | 具体的な次アクションに置き換える |

## 文書生成時チェック
1. 冒頭に結論があるか→なければ移動
2. コスト・期間・効果が抜けていたら「[数値を確認]」プレースホルダー
3. 禁止表現を照合→代替提示
4. 宛先を判定→相手別スタイル適用
5. A4半ページ超→「詳細は別添え」で圧縮
6. 推量・受け身・謝罪過多→断定調に書き換え
