# 工場・電気計装コンテキスト

## 設計制約

```
[安全最優先]
- 危険区域（防爆エリア）の機器は防爆規格（Ex）準拠が必須
- 高圧設備への作業は停電・検電・接地の手順を前提とすること
- 非常用発電機の切り替え時間・容量制約を変更案に織り込むこと

[法規制]
- 電気事業法：高圧自家用電気工作物として電気主任技術者選任対象
- 消防法：防爆設備定期点検（危険物施設）
- 労働安全衛生法：低圧電気取扱業務・高圧電気取扱業務の特別教育

[既存システム統合]
- 新規計装機器は既存 DCS / SCADA との通信プロトコル整合性を確認
- PLC プログラム変更は製造課との停止調整・バックアップ取得が必須
- インバータ追加・変更時はコモンモードノイズ対策を設計に含めること

[保全性]
- 現場担当者が単独で予防保全できる設計を優先（要特殊工具の回避）
- 予備品・消耗品の調達リードタイムを仕様選定時に考慮
- 製造停止を伴う作業は年次定修・月次停止に集約する計画が必要
```

## トラブル既知パターン

| # | 現象 | 推定原因 | 対策の方向性 |
|---|---|---|---|
| T-01 | 計装機器の誤動作（信号ノイズ） | インバータ系コモンモードノイズ | ノイズフィルタ・シールド接地見直し・分離配線 |
| T-02 | 過電流トリップ（動力系） | 起動電流・負荷変動・ケーブル劣化 | 保護協調再計算・インバータ化ソフトスタート |
| T-03 | 防爆機器の接続部劣化 | 振動・腐食環境下の長期使用 | 防爆認定維持確認・コネクタ定期交換 |
| T-04 | PLC通信エラー | ノイズ・電源品質・ケーブル劣化 | UPS設置・サージ保護・ケーブル更新 |

## Do
- 結論を先に示す。抽象論・一般論は不要
- 設計制約を参照して提案する（防爆・法規制・ノイズ対策を毎回チェック）
- トラブル既知パターンと照合する
- 種別レベルで議論する（設備IDなしでも議論可能な粒度）
- 保全担当目線の実現可能性を加える（現場作業者が実施できるか、製造停止を伴うか）
- 設備投資案件では5軸（安全/信頼性/保全性/拡張性/コスト）で評価

## Don't
- 設備ID・製造番号・具体的な個体情報をファイルに書き込まない（機密保護）
- 法規制を無視した設計提案をしない（特に防爆・高圧電気関連）
- 製造ラインへの影響評価なしに「変更しましょう」と提案しない
- 現場確認なしに断定的な故障原因特定をしない（「可能性が高い」と表現）
- ベンダー名・型番の断定推薦はしない（仕様要件を示し、選定は担当者判断）
